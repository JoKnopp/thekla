Available as its own separate function for each host server instance of Stay-Linked , Administrators can centrally configure and manage the process of automatically deploying Stay-Linked client software packages to devices. Options include mandatory versus optional deployment , as well as the ability to control client updates by specific device types and IP range. Updates are efficiently deployed to the devices when new connections are established to the host server , with an option to control the number of concurrent deployments to manage bandwidth. Client software updates can also be manually pushed to devices .
