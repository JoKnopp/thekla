Reboot target workstation and check event logs for installation failure messages. You ( as an administrator ) can use Group Policy to assign or to publish software to users or computers in a domain. Additionally , it is useful to be able to deploy software based on group membership. A Group Policy object ( GPO ) is usually applied only to members of an organizational unit ( OU ) to which the GPO is linked. Because a user can not be located in several OUs at the same time , you must be able to apply Group Policy settings outside the boundaries of OUs. This article describes how to have your software deployment policy applied to users who are not in an OU .
